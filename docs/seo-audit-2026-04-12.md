# SEO Audit: Public SEO Pages

Date: 2026-04-12

## Executive Summary

The SEO estate is already strong. The biggest opportunities are not "write 1,000 more pages" or "rewrite everything". The biggest opportunities are:

- clean up route and sitemap drift
- reduce cannibalisation across overlapping pages
- strengthen hub-and-spoke structure around Elec-Mate's commercial pillars
- bring the manual SEO pages closer to the stronger template system
- make the pages feed product pages and signup flows more deliberately

This should be treated as an acquisition system, not just a content library.

## Main Issues

### 1. Route duplication exists

There are `8` duplicate route paths in `src/routes/SEORoutes.tsx`.

Duplicates found:

- `/guides/appendix-4-tables-bs-7671`
- `/guides/building-regulations-electrical`
- `/guides/eicr-for-commercial-premises`
- `/guides/going-self-employed-electrician`
- `/guides/heat-pump-electrical-requirements`
- `/guides/insulation-resistance-minimum-values`
- `/guides/napit-certificate-guide`
- `/guides/prospective-fault-current-explained`

This is the highest-priority technical cleanup.

### 2. Sitemap overlap is too heavy

`public/sitemap-seo.xml` overlaps heavily with the specialist sitemaps:

- overlaps `59` URLs with `sitemap-tools.xml`
- overlaps `261` URLs with `sitemap-guides.xml`
- overlaps `22` URLs with `sitemap-training.xml`
- overlaps `4` URLs with `sitemap-compare.xml`

This is not fatal, but it is messy and weakens sitemap clarity. The sitemap strategy should be simplified so each URL lives in one logical sitemap only.

### 3. URL architecture is mixed

There is a good structured taxonomy for:

- `/guides`
- `/tools`
- `/training`
- `/compare`
- `/electricians`

But `332` pages still live at the root. Many of these are valid pages, but the root-level sprawl makes the estate feel less like a controlled topic system.

High-volume root-level clusters include:

- `consumer-unit-replacement-*`: `12`
- `ev-charger-installation-*`: `12`
- `landlord-electrical-safety-*`: `10`
- `eicr-*`: `22`
- `electrician-*`: `20`

These likely need clearer grouping, stronger hub pages, and more deliberate internal linking.

### 4. The manual pages are the weak point

The `54` `PublicPageLayout` pages are the likely weak cluster.

Compared with the shared templates, these pages do not consistently expose:

- `relatedPages=`
- `faqs=`
- `datePublished=`
- `dateModified=`

Many of these pages are still strong individually, but this is the area where consistency is weakest and the easiest place for SEO quality to drift.

### 5. Cannibalisation risk exists

Examples of likely overlap:

- `NAPICCertificateGuidePage` vs `NAPICertificateGuidePage`
- `PolarityTestGuidePage` vs `PolarityTestingGuidePage` vs `PolarityTestingPage`
- `EICRForCommercialPage` vs `EICRForCommercialPremisesPage`
- `AppendixFourTablesPage` vs `AppendixFourTablesGuidePage`
- `OffJobTrainingGuidePage` vs `OffJobTrainingHoursPage`
- `ProspectiveFaultCurrentGuidePage` vs `ProspectiveFaultCurrentCalculatorPage`

Not every pair should be merged. But each needs a clear role:

- informational guide
- tool page
- comparison page
- location page
- commercial landing page

If two pages target the same search intent with the same commercial angle, one will dilute the other.

## Commercial SEO Read

The pages are currently strongest as a knowledge library. To work harder for Elec-Mate, each cluster needs to feed one of the commercial pillars:

- certificates
- calculators
- inspection and testing
- apprentice training
- AI tools
- quoting / invoicing
- compliance / RAMS

Each content cluster should behave like:

`traffic page -> hub page -> money page -> signup`

Examples:

- fault-finding guides -> calculator pages -> AI diagnosis tools -> signup
- EICR educational pages -> certificate pages -> inspection/testing flows -> signup
- apprenticeship guides -> training hub -> course/app pages -> signup
- business guides -> quoting/invoicing/RAMS pages -> signup

Right now the ingredients exist, but the linking strategy needs to become more deliberate.

## Priority Recommendations

### Priority 1: Technical cleanup

- remove the `8` duplicate route paths
- simplify sitemap strategy so URLs do not appear in multiple sitemap files
- verify each canonical target matches the intended public URL
- audit `EICRCertificatePage.tsx` because it is the lone page outside the standard template families

### Priority 2: Audit and upgrade the 54 manual pages

Start with the `PublicPageLayout` pages because they are the highest-leverage improvement area.

Bring them closer to template quality where appropriate:

- clearer freshness signals
- stronger related-page blocks
- better FAQ consistency
- more deliberate internal bridges into Elec-Mate product pages
- more explicit hub-page linking

### Priority 3: Build a true cluster strategy

Define hub pages for the main acquisition pillars:

- certificates
- calculators
- inspection/testing
- apprentice training
- AI tools
- electrician business tools

Then make surrounding pages reinforce those hubs instead of linking sideways at random.

### Priority 4: Resolve cannibalisation

For overlapping pages, decide one of:

- keep both, but separate intent clearly
- merge and redirect
- keep one as the main page and demote the other to supporting internal content

### Priority 5: Improve root-level SEO architecture

The root-level pages should be reviewed cluster by cluster.

Likely actions:

- retain high-value direct slugs where brand/search behavior justifies it
- add better parent hubs for city/service clusters
- standardise internal links between service pages and product pages
- reduce orphan-like pages that sit at root without strong contextual support

## Pages / Clusters To Review First

### First cluster: manual pages

Examples:

- AI pages
- calculator landers
- certificate pages
- training landing pages
- comparison pages built manually

Reason: these are closest to commercial intent and easiest to improve quickly.

### Second cluster: overlapping guide families

Examples:

- NAPIC / NICEIC / scheme content
- EICR commercial content
- polarity testing content
- appendix / regs / guide duplicates

Reason: these are the clearest cannibalisation risks.

### Third cluster: local/service root-level pages

Examples:

- `consumer-unit-replacement-*`
- `ev-charger-installation-*`
- `landlord-electrical-safety-*`
- `eicr-*`
- `electrician-*`

Reason: these clusters can drive visibility, but only if they have clear hubs and do not become thin, isolated landing pages.

## Suggested Rollout

### Phase 1

- fix duplicate routes
- clean sitemap overlap strategy
- audit the 54 manual pages

### Phase 2

- define hub pages and internal-link rules for the main commercial clusters
- map every cluster to its Elec-Mate money page

### Phase 3

- resolve cannibalisation cases
- improve cluster architecture for root-level service pages

### Phase 4

- review content depth and conversion quality on the top-traffic / top-opportunity pages only

## Bottom Line

The SEO strategy is already viable. The site does not look weak. The next level is not more page volume. The next level is:

- cleaner SEO architecture
- better cluster ownership
- clearer commercial funnels
- fewer overlapping pages
- stronger alignment between educational traffic and Elec-Mate's product pages

That is how the SEO pages start to work for Elec-Mate instead of just existing on Elec-Mate.
