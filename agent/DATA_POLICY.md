## 5. User Rights (UK GDPR)

The electrician has the following rights, exercisable at any time:

### Right of Access

- "What do you know about me?" → agent reads MEMORY.md and preferences, presents everything
- Full data export available from app settings (JSON/CSV)
- Activity log viewable in-app at all times

### Right to Rectification

- "That's wrong, I charge £350 not £320" → agent updates immediately
- All preferences editable via agent conversation or app settings

### Right to Erasure ("Right to be Forgotten")

- "Forget everything about me" → full memory wipe
- "Forget that" → delete specific memory entry
- Account deletion → all data deleted within 30 days (including archives)
- Agent workspace deleted immediately on deprovisioning

### Right to Restrict Processing

- "Stop" / "Pause" → agent stops all outbound actions immediately
- Can restrict specific rail processing: "Don't chase invoices automatically"
- Can restrict specific data access: "Don't look at my calendar"

### Right to Data Portability

- Export all agent data (conversations, preferences, activity log) as JSON
- Available via app settings or by contacting support

### Right to Object

- Can object to any automated processing
- Can opt out of any rail individually
- Can disable the agent entirely and retain all Elec-Mate app access

### Rights Regarding Automated Decision-Making

- The agent does NOT make fully automated decisions with legal or significant effects
- All financial actions, client communications, and business decisions require human approval
- The electrician can override any agent suggestion

## 6. Client Data (Data Subjects Who Are NOT Users)

### Electricians' Clients

The electrician's clients are data subjects whose data the agent processes on the electrician's behalf.

| Data                        | Source                                                  | Purpose                                  |
| --------------------------- | ------------------------------------------------------- | ---------------------------------------- |
| Name, email, phone, address | Entered by electrician in Elec-Mate                     | Job management, invoicing, cert delivery |
| WhatsApp messages           | Client sends message to agent                           | Enquiry handling, booking                |
| Payment history             | Stripe payment events                                   | Invoice tracking                         |
| Client portal access        | Portal view timestamps, pages accessed                  | Service delivery                         |
| Digital signatures          | DocuSign envelope ID, signature timestamps, audit trail | Contract performance                     |

### Electrician's Responsibilities

The electrician is the **data controller** for their clients' personal data. Elec-Mate (and by extension, the agent) is the **data processor**.

The electrician must:

- Have lawful basis for holding client data (contractual relationship)
- Obtain WhatsApp consent before the agent messages a client (ELE-158)
- Honour opt-out requests from clients immediately
- Not ask the agent to send marketing messages without explicit client consent

### Client Opt-Out

If a client sends "STOP" or requests removal:

1. Agent immediately flags client as `no_contact`
2. Client removed from all future outreach rails
3. Agent confirms to electrician: "[Client] has opted out of messages"
4. Existing data retained for legal/compliance purposes (cert records, invoice records) but no further outbound contact

## 8. Security Incidents

### If A Data Breach Occurs

1. Agent processing halted immediately (kill switch)
2. Elec-Mate engineering team notified within 1 hour
3. Affected users notified within 72 hours (per UK GDPR Article 33)
4. ICO notified within 72 hours if breach poses risk to individuals
5. Full incident report compiled and shared with affected users
6. Remediation steps implemented before processing resumes

### Breach Notification Content

- What happened
- What data was affected
- What Elec-Mate is doing about it
- What the user should do
- Contact details for further information

## 10. Agent-Specific Data Rules

### The Agent Must

- Log every data access to the activity trail
- Never cache client personal data beyond the current conversation context
- Never store payment card details (Stripe handles all payment data)
- Never transmit data to any endpoint not listed in this policy
- Delete conversation context older than 24 hours (compact to MEMORY.md only)
- Respect all user preference overrides immediately

### The Agent Must Not

- Infer or guess personal data it doesn't have
- Cross-reference data between different users
- Store biometric data (voice prints from transcription are not stored)
- Use client data for any purpose other than the electrician's direct instructions
- Retain data after the user has requested deletion
- Include mental health data (mood check-ins, journal entries) in morning briefings, weekly digests, or any automated output — this data is only accessible when the user explicitly asks for it
- Share safety incident data with third parties without anonymisation and explicit opt-in — anonymised data may be used for industry-wide safety alerts only if the user has opted in
- Include expense receipt images in any report or export beyond the expense record itself
